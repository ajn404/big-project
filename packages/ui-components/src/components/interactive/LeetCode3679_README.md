# LeetCode 3679 Visualizer - 使库存平衡的最少丢弃次数

## 组件描述

这是一个交互式可视化组件，用于演示 LeetCode 3679 题目的滑动窗口算法。该组件使用 D3.js 来可视化算法执行过程。

## 功能特性

- 🎯 **实时可视化**: 显示滑动窗口在数组上的移动过程
- 📊 **D3.js 图表**: 使用 D3.js 绘制清晰的可视化图表
- ⚙️ **自定义参数**: 支持自定义 arrivals 数组、窗口大小 w 和最大出现次数 m
- 🎮 **交互控制**: 单步执行、自动执行、暂停、重置等控制功能
- 🎨 **直观显示**: 
  - 绿色圆圈：保留的物品
  - 红色圆圈：丢弃的物品
  - 灰色圆圈：未处理的物品
  - 蓝色区域：当前滑动窗口

## 参数说明

- `arrivals`: 到达的物品类型数组，例如 `[1,2,1,3,2,1]`
- `w`: 滑动窗口大小，表示考虑的天数范围
- `m`: 每种类型在窗口中最多出现的次数

## 算法逻辑

1. 对于第 i 天到达的物品，考虑窗口范围 `[max(1, i - w + 1), i]`
2. 检查该物品类型在窗口内已保留的次数
3. 如果保留该物品会导致次数超过 m，则必须丢弃
4. 否则可以保留该物品
5. 统计总的丢弃次数

## 示例

默认示例：
- arrivals: `[1,2,1,3,2,1]`
- w: `3` (窗口大小为3天)  
- m: `2` (每种类型最多出现2次)

执行过程：
- Day 1: 物品1 → 保留 (窗口内物品1出现1次 ≤ 2)
- Day 2: 物品2 → 保留 (窗口内物品2出现1次 ≤ 2)
- Day 3: 物品1 → 保留 (窗口[1,3]内物品1出现2次 ≤ 2)
- Day 4: 物品3 → 保留 (窗口[2,4]内物品3出现1次 ≤ 2)
- Day 5: 物品2 → 保留 (窗口[3,5]内物品2出现1次 ≤ 2)
- Day 6: 物品1 → 丢弃 (窗口[4,6]内物品1已出现1次，再保留会超过2次)

最终丢弃数量：1
